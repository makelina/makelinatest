# Тест-кейс для оплаты картой

## Введение

Цель этого тест-кейса — проверить функциональность оплаты с помощью карт в системе. Он охватывает различные сценарии, чтобы обеспечить надежность и безопасность процесса оплаты.

## Предусловия

1. Пользователь должен быть зарегистрирован и авторизован в системе.
2. На счету пользователя должна быть доступная сумма для оплаты.
3. Система должна поддерживать основные типы карт: Visa, MasterCard, American Express.

## Сценарии тестирования

### 1. Успешная оплата

- **Шаги**:
  1. Перейдите на страницу оплаты.
  2. Введите корректные данные карты.
  3. Подтвердите оплату.

- **Ожидаемый результат**: Оплата проходит успешно, пользователь получает подтверждение.

### 2. Оплата отклонена из-за недостатка средств

- **Шаги**:
  1. Перейдите на страницу оплаты.
  2. Введите данные карты с недостаточным балансом.
  3. Подтвердите оплату.

- **Ожидаемый результат**: Оплата отклонена, пользователь получает уведомление о недостатке средств.

### 3. Неправильные данные карты

- **Шаги**:
  1. Перейдите на страницу оплаты.
  2. Введите некорректные данные карты (например, неверный номер или срок действия).
  3. Подтвердите оплату.

- **Ожидаемый результат**: Оплата отклонена, пользователь получает сообщение об ошибке в данных карты.

## Постусловия

- Проверить, что транзакция зарегистрирована в системе.
- Убедиться, что пользователь уведомлен о статусе транзакции.
